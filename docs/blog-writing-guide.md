# ブログ記事作成ガイド

## 記事の仕様

| 項目 | 内容 |
|------|------|
| 文字数 | 約3000文字（HTML本文） |
| 形式 | HTML（microCMSリッチエディタ互換） |
| サムネイル | Gemini で生成（1200×630px推奨） |
| 事実情報 | 複数サイトを確認して正確に記載 |
| 参考書 | 具体的な書名・出版社・特徴を記載 |

---

## 手順

### Step 1: 記事本文を作成（Claude と対話）

以下のプロンプトをClaudeに送って記事を作成する。
`{テーマ}` と `{カテゴリ}` を書き換えて使う。

```
あなたは大学受験の英語指導に精通したプロのライターです。
以下の条件でブログ記事を1本書いてください。

【テーマ】
{テーマ}
例: 近大英語の長文読解で8割取るための勉強法

【カテゴリ】
{カテゴリ}
※ 近大英語 / 勉強法 / お知らせ のいずれか

【記事の要件】
- 文字数: 約3000文字
- 対象読者: 近畿大学を志望する受験生（高2〜高3）
- 文体: です・ます調。親しみやすく、でも信頼感のあるトーン
- 出力形式: HTML（h2, h3, p, ul, li, strong, a タグを使用）

【必須で含める内容】
1. 具体的な試験情報（試験時間・配点・問題数など）
   → 必ず近畿大学の公式情報や信頼できるサイトの情報を元にし、
     「近畿大学公式サイトによると…」のように出典を明記すること
2. 具体的な参考書名を3冊以上挙げる
   → 書名・出版社・特徴・どのレベルの人向けかを書く
   → 例:「『システム英単語』（駿台文庫）— 共通テスト〜中堅私大レベル」
3. 具体的な学習スケジュールや時間配分の目安
4. 記事の最後に、ドリレオの近大英語ドリルへの自然な誘導
   → <a href="/kindai">近大英語 長文読解ドリル</a> へのリンクを含める

【構成の目安】
- 導入（なぜこのテーマが重要か）
- 本論（3〜5セクション、h2で区切る）
- まとめ + ドリレオへの誘導

【出力】
以下の3点をそれぞれ出力してください:
1. title: 記事タイトル（30〜40文字、SEOを意識）
2. description: meta description用の要約（80〜120文字）
3. content: 記事本文（HTML）
```

**対話のコツ:**
- 1回目の出力を読んで「○○の部分をもっと具体的に」「参考書の説明を増やして」等フィードバックする
- 事実情報（試験時間60分・100点満点等）は自分でも確認する
- 納得いくまで何往復かやり取りする

---

### Step 2: サムネイル画像を作成（Gemini）

以下のプロンプトをGeminiの画像生成に送る。`{テーマ}` を書き換えて使う。

```
以下のブログ記事のサムネイル画像を作成してください。

【記事テーマ】{テーマ}
【サイト名】ドリレオ
【要件】
- サイズ: 1200×630px（OGP比率）
- テイスト: 清潔感のある青系（#2563EB）ベース
- 日本語テキストは入れない（文字化けするため）
- 大学受験・英語学習を連想させるイラストやアイコン
- シンプルで視認性が高いデザイン
```

生成した画像をmicroCMSの記事のサムネイルフィールドにアップロードする。

---

### Step 3: microCMS に投稿

#### 方法A: 管理画面から手動投稿

1. https://{サービスID}.microcms.io にログイン
2. 「ブログ」→「追加」
3. 各フィールドを入力:
   - **タイトル**: Step 1 で作成した title
   - **スラッグ**: URLに使う英語文字列（例: `kindai-reading-tips`）
   - **説明文**: Step 1 で作成した description
   - **本文**: Step 1 で作成した content（HTMLをリッチエディタに貼り付け）
   - **サムネイル**: Step 2 で作成した画像をアップロード
   - **カテゴリ**: 近大英語 / 勉強法 / お知らせ から選択
4. 「公開」

#### 方法B: スクリプトで投稿

`scripts/seed-blogs.mjs` の `blogs` 配列を書き換えて実行:

```bash
node scripts/seed-blogs.mjs
```

※ サムネイルはスクリプトでは設定できないので、投稿後にmicroCMS管理画面から追加する。

---

### Step 4: 確認

- [ ] `/blog` に記事カードが表示される
- [ ] `/blog/{slug}` で記事全文が表示される
- [ ] PC版で2カラムレイアウトになっている
- [ ] ページソースに Article JSON-LD がある
- [ ] OGP画像が設定されている（SNSシェアデバッガーで確認）
- [ ] Search Console でインデックス登録をリクエスト
